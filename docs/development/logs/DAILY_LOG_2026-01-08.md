### 问题 1: 操作按钮图标尺寸问题

#### 问题描述
- **预期**: 32×32px 的可点击区域，18px 图标
- **实际**: 图标渲染为 4.67×6px，完全不可用
- **影响**: 用户无法点击操作按钮，严重损害用户体验

#### 调试过程

**第一次尝试**（test-case-workbench.tsx:335-364）:
```tsx
<Button
  className="h-8 w-8 min-w-[32px] p-0 text-slate-400 hover:bg-[#F1F5F9] hover:text-[#10B981] rounded-full transition-all duration-200"
  style={{ minWidth: '32px', minHeight: '32px' }}
>
  <Play className="h-[18px] w-[18px]" strokeWidth={2.5} style={{ width: '18px', height: '18px' }} />
</Button>
```

**问题**: ❌ 样式未生效，图标依然显示为 4.67×6px

**第二次尝试**（增加 padding）:
```tsx
<Button
  className="h-8 w-8 min-w-[32px] p-2 ..."  // 从 p-0 改为 p-2
  style={{ minWidth: '32px', minHeight: '32px' }}
>
```

**问题**: ❌ 仍然无效，尺寸没有改变

#### 根本原因分析

通过检查 `components/ui/button.tsx`，发现问题所在：

```tsx
// Button 组件的默认样式
const Button = React.forwardRef<HTMLButtonElement, ButtonProps>(
  ({ className, variant = 'default', size = 'default', ...props }, ref) => {
    return (
      <button
        className={cn(
          'inline-flex items-center justify-center rounded-md font-medium transition-colors focus:outline-none focus:ring-2 focus:ring-blue-500 focus:ring-offset-2 disabled:opacity-50 disabled:pointer-events-none',
          // ... variant styles
          {
            'h-9 px-3 text-sm': size === 'sm',  // ❌ 这里强制设置了 h-9 (36px)
          },
          className  // 我们的 className 在这里
        )}
        ref={ref}
        {...props}
      />
    )
  }
)
```

**关键发现**:
1. `size="sm"` 会应用 `h-9` class（Tailwind的 `height: 2.25rem` = 36px）
2. Tailwind CSS 的 class 优先级高于普通 inline styles
3. 我们的内联 `style={{ width: '32px', height: '32px' }}` 被 `h-9` 覆盖

#### 最终解决方案

**使用 `!important` 强制覆盖所有 CSS**:

```tsx
<Button
  variant="ghost"
  size="sm"
  onClick={() => onRun(testCase)}
  className="text-slate-400 hover:text-[#10B981] rounded-full transition-all duration-200"
  title="运行"
  style={{
    width: '32px !important',
    height: '32px !important',
    minWidth: '32px !important',
    maxWidth: '32px !important',
    display: 'flex',
    justifyContent: 'center',
    alignItems: 'center',
    padding: '0 !important'
  }}
>
  <Play strokeWidth={2.5} style={{
    width: '18px',
    height: '18px',
    minWidth: '18px',
    maxWidth: '18px'
  }} />
</Button>
```

**关键点**:
- ✅ 使用 `!important` 强制覆盖 Tailwind 的 `h-9` class
- ✅ 移除所有 `className` 中的尺寸类（`h-8`, `w-8`, `h-[18px]`, `w-[18px]`）
- ✅ 图标也使用 inline style（不带 `!important`，因为没有冲突）
- ✅ 设置 `minWidth` 和 `maxWidth` 确保图标不会被缩放

**影响文件**:
- `tarsight-dashboard/components/test-case-workbench.tsx` (行 349-378)

### 问题 2: 优先级徽章颜色混淆

#### 问题描述
- **P0（关键）** 和 **P2（普通）** 看起来都是蓝色，无法区分
- **原因**: 之前的 `getPriorityBadge()` 函数返回 className，但可能有其他样式覆盖

#### 调试过程

**第一次尝试**（使用 className）:
```tsx
function getPriorityBadge(level: string) {
  const badges = {
    P0: 'bg-[#FEE2E2] text-[#991B1B] border-transparent font-semibold rounded-lg',
    P1: 'bg-[#FFEDD5] text-[#9A3412] border-transparent font-semibold rounded-lg',
    P2: 'bg-[#DBEAFE] text-[#1E40AF] border-transparent font-semibold rounded-lg',
    P3: 'bg-[#F3F4F6] text-[#374151] border-transparent font-semibold rounded-lg'
  }
  return badges[level as keyof typeof badges] || badges.P2
}

// 使用
<Badge className={`text-xs px-2.5 py-1 ${getPriorityBadge(testCase.level || 'P2')}`}>
  {testCase.level || 'P2'}
</Badge>
```

**问题**: ❌ 颜色没有生效，或者被其他样式覆盖

#### 最终解决方案

**改用 inline style 对象**:

```tsx
// 1. 修改函数返回 style 对象
function getPriorityStyle(level: string) {
  const styles: Record<string, { backgroundColor: string; color: string }> = {
    P0: { backgroundColor: '#FEE2E2', color: '#991B1B' },
    P1: { backgroundColor: '#FFEDD5', color: '#9A3412' },
    P2: { backgroundColor: '#DBEAFE', color: '#1E40AF' },
    P3: { backgroundColor: '#F3F4F6', color: '#374151' }
  }
  return styles[level as keyof typeof styles] || styles.P2
}

// 2. 使用 <span> 而不是 <Badge>，应用 style
<span
  style={priorityStyle}
  className="px-2.5 py-0.5 rounded-full font-semibold text-xs inline-block"
>
  {testCase.level || 'P2'}
</span>
```

**颜色映射**:
- **P0（关键）**: 背景 `#FEE2E2`（浅红），文字 `#991B1B`（深红）
- **P1（重要）**: 背景 `#FFEDD5`（浅橙），文字 `#9A3412`（深橙）
- **P2（普通）**: 背景 `#DBEAFE`（浅蓝），文字 `#1E40AF`（深蓝）
- **P3（低）**: 背景 `#F3F4F6`（浅灰），文字 `#374151`（深灰）

**影响文件**:
- `tarsight-dashboard/components/test-case-workbench.tsx` (行 22-29, 311-316)

### 问题 3: 网格列宽失衡

#### 问题描述
- **用例名称列**过窄（1.2fr），长名称被截断
- **ID和优先级列**过宽（80px, 100px），浪费空间
- **视觉不协调**，列与列之间间距过大

#### 最终解决方案

**使用 inline style 强制应用网格布局**:

```tsx
// 表头

  className="grid gap-3 px-4 py-2 bg-slate-50 border-b border-slate-100"
  style={{ gridTemplateColumns: '70px 90px 1.5fr 80px 1fr 120px' }}

  {/* ... 列标题 ... */}
</div>

// 表行
<div
  className="group grid gap-3 px-4 py-3 hover:bg-[#F8FAFC] transition-all duration-200 border-b border-slate-50 last:border-0 items-center"
  style={{ gridTemplateColumns: '70px 90px 1.5fr 80px 1fr 120px' }}
>
  {/* ... 行内容 ... */}
</div>
```

**列宽调整**:
- **ID**: 70px（减少 10px）
- **优先级**: 90px（减少 10px）
- **用例名称**: 1.5fr（增加 0.3fr）
- **方法**: 80px（保持不变）
- **URL**: 1fr（保持不变）
- **操作**: 120px（保持不变）

### 问题 4: 按钮文字挤压

#### 问题描述
- 标签输入框的"添加"按钮文字换行
- 断言规则的"添加规则"按钮文字换行
- 原因：flex 容器空间不足时，按钮被压缩

#### 最终解决方案

**添加 `flex-shrink: 0` 防止按钮收缩**:

```tsx

<Button
  type="button"
  onClick={addTag}
  variant="outline"
  size="sm"
  className="whitespace-nowrap min-w-[60px] flex-shrink-0 transition-all duration-200"
  //                                              ^^^^^^^^^^^^ 关键
>
  添加
</Button>

// 断言规则的"添加规则"按钮
<Button
  type="button"
  onClick={() => { /* ... */ }}
  variant="outline"
  size="sm"
  className="text-xs whitespace-nowrap min-w-[80px] flex-shrink-0 border-[#3B82F6] text-[#3B82F6] hover:bg-[#DBEAFE] hover:text-[#1E40AF] transition-all duration-200"
  //                                              ^^^^^^^^^^^^ 关键

  <Plus className="h-3 w-3 mr-1" />
```

**关键点**:
- ✅ `flex-shrink: 0` 防止按钮在 flex 容器中被压缩
- ✅ `whitespace-nowrap` 防止文字换行
- ✅ `min-w-[60px]` 或 `min-w-[80px]` 设置最小宽度

## 💡 核心经验教训

### 1. CSS 优先级问题

#### 问题层级（从低到高）:
1. **浏览器默认样式**
2. **Tailwind CSS classes**（如 `h-9`, `w-8`）
3. **自定义 CSS classes**（如 `.custom-button`）
4. **inline styles**（如 `style={{ width: '32px' }}`）
5. **inline styles with `!important`**（如 `style={{ width: '32px !important' }}`）✅ 最高优先级

#### 关键发现:
- **Tailwind classes 优先级高于普通 inline styles**
- **React 组件内部的 className 合并可能导致意外的优先级问题**
- **`!important` 是最后的武器，但在对抗第三方组件时非常有效**

#### 最佳实践:
```tsx
// ❌ 不推荐：混合使用 className 和 style
<Button className="h-8 w-8" style={{ width: '32px' }}>

// ✅ 推荐：要么全部用 className
<Button className="h-8 w-8 min-w-[32px]">

// ✅✅ 最强：当需要覆盖第三方组件时，用 !important
<Button style={{ width: '32px !important', height: '32px !important' }}>
```

---

### 2. React 组件样式覆盖策略

#### 场景 1: 自己的组件
```tsx
// ✅ 直接修改组件的 className
<MyButton className="custom-size">
```

#### 场景 2: 第三方组件（如 shadcn/ui）
```tsx
// ✅ 使用 !important 强制覆盖
<Button
  style={{
    width: '32px !important',
    height: '32px !important',
    minWidth: '32px !important',
    maxWidth: '32px !important'
  }}
>
```

#### 场景 3: 需要动态样式
```tsx
// ✅ 使用 style 对象而不是 className
function getPriorityStyle(level: string) {
  return {
    backgroundColor: level === 'P0' ? '#FEE2E2' : '#DBEAFE',
    color: level === 'P0' ? '#991B1B' : '#1E40AF'
  }
}

<span style={getPriorityStyle(level)}>
```

### 4. Flexbox 布局的常见陷阱

#### 问题: 按钮文字换行
```tsx
// ❌ 错误：没有防止收缩
<div className="flex gap-2">
  <Input className="flex-1" />
  <Button>添加</Button>  {/* 当空间不足时会换行 */}
</div>
```

```tsx
// ✅ 正确：防止收缩
<div className="flex gap-2">
  <Input className="flex-1" />
  <Button className="flex-shrink-0 whitespace-nowrap min-w-[60px]">
    添加
  </Button>
</div>
```

#### 关键属性:
- `flex-shrink: 0` - 防止元素收缩
- `whitespace-nowrap` - 防止文字换行
- `min-w-[Xpx]` - 设置最小宽度

---

### 5. Grid 布局的稳定性

#### 问题: Tailwind 的 grid 类可能被覆盖
```tsx
// ❌ 可能被覆盖
<div className="grid grid-cols-[70px_90px_1.5fr_80px_1fr_120px]">
```

```tsx
// ✅ 更稳定
<div className="grid" style={{ gridTemplateColumns: '70px 90px 1.5fr_80px_1fr_120px' }}>
```

#### 何时使用 inline style:
- ✅ 需要精确控制列宽比例
- ✅ 使用了 fr 单位（弹性比例）
- ✅ 发现 Tailwind class 被覆盖时
