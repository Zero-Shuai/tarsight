# AI 上下文快速参考

## 📌 重要提示
在新对话开始时，告诉 AI：

```
请先阅读以下文件了解项目上下文：
1. docs/development/logs/ - 查看最新的每日开发日志
2. docs/INDEX.md - 查看文档索引
3. docs/development/README.md - 了解开发文档结构
```

## 🗂️ 文档分类

### 开发日志 (docs/development/logs/)
- **命名**: `DAILY_LOG_YYYY-MM-DD.md`
- **内容**: 每日完成的工作、问题、解决方案
- **用途**: 了解最近的工作进展

### 实现文档 (docs/development/)
- 功能实现的详细记录
- 设计决策和演变过程
- 废弃方案的记录（避免重复尝试）

### 归档文件 (supabase_version/archive/)
- 不再使用的临时脚本
- 保留原因：作为历史记录

## 💡 快速查找

### 我想知道...
- **最近做了什么**: 查看 `docs/development/logs/` 最新的日志
- **某个功能怎么实现的**: 查看 `docs/development/` 对应的文档
- **项目整体架构**: 查看 `docs/architecture/`
- **遇到问题怎么办**: 查看 `docs/troubleshooting/`

## 📝 添加新内容

### 每日日志
1. 在 `docs/development/logs/` 创建 `DAILY_LOG_YYYY-MM-DD.md`
2. 记录当天完成的工作
3. 遇到的问题和解决方案
4. 待办事项

### 功能文档
1. 在 `docs/development/` 创建对应的文档
2. 记录需求、设计、实现过程
3. 包含代码示例和截图

### 归档文件
1. 移动到 `supabase_version/archive/`
2. 在 `archive/README.md` 中说明用途
